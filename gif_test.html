Testing GIF Support in Music Blocks

This is a test page to verify that GIF support is working properly in Music Blocks.

Steps to test:

  1. Go to Music Blocks
  2. Drag a "Media" block from the Media palette
  3. Click on the Media block to open the file picker
  4. Select a GIF file
  5. Connect the Media block to a "Show" block
  6. Run the program to verify the GIF displays correctly

Expected behavior:

  • The file picker should accept GIF files
  • The Media block should show a thumbnail of the GIF
  • The Show block should display the GIF on the canvas
  • For animated GIFs, the first frame should be displayed (note: CreateJS typically shows only the first frame of animated GIFs)
  • No console errors should appear

Changes made:

  • Updated HTML file input to explicitly accept GIF files: accept="image/*,.gif,.png,.jpg,.jpeg,.svg,.webp"
  • Added error handling to image loading in block.js, turtle.js
  • Updated help text to mention GIF support
  • Added console logging for debugging image loading issues

Notes:

If you encounter issues:

  • Check the browser console for error messages
  • Ensure the GIF file is not corrupted
  • Try with a small, simple GIF file first
  • Animated GIFs will typically display only the first frame due to CreateJS limitations
